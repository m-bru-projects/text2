## Core UI concept
Three zones: 

1. **Main text area**

   * Central focus of the screen.
   * Displays location descriptions, dialogue, events, responses and system feedback.
   * Should feel literary, atmospheric and readable.
   * Use strong typography, generous line height and a comfortable text width.
   * Subtly highlight important nouns and events, but keep this restrained.

2. **Right panel**

   * Shows general progress
   * Should feel like a hand-drawn map, casebook, field notebook or annotated archive.
   * Include tabs:

     * Map
     * Story
     * People
     * Places
     * Knowledge items gained
     * Inventory

   * The map can be node-based rather than literal.
   * Locations not visited should be invisible
   * Keep this panel supportive, not dominant.


## Interaction model

Use a **text-first interaction model**.

Preferred interaction options:

* A command input at the bottom:
  `What do you do?`
* Only general suggested commands, no context specific ones
* The player can type commands.
* The game can offer autocomplete, command history or gentle suggestions.
* Avoid “button menu adventure” feeling.

## Visual style

The mood should be:

* dark
* literary
* investigative
* eerie
* refined
* modern, not cheesy fantasy parchment

Suggested visual language:

* charcoal / deep blue-black background
* warm amber accent color
* soft old-paper or notebook surfaces for panels
* subtle grain/noise texture
* fine etched divider lines
* serif font for narrative text
* clean sans-serif for UI labels
* faint rain, dust or amber pulse animations
* restrained transitions

Avoid:

* bright fantasy RPG UI
* heavy parchment cliché
* large game buttons
* excessive icons
* cluttered HUD
* visible numeric stat overload

## Top bar

Include a subtle status bar with current context, for example:

`UNDERBRIDGE GATE · FIRST NIGHT · RAIN · THE LETTER IS WARM`

This should show:

* current location
* time/phase
* weather or atmosphere
* one or two important qualitative states

Avoid numeric stat bars unless absolutely necessary.


## UX priorities

The UI should help the player manage complexity without breaking immersion.
Not too much handholding.
